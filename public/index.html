Messages are encrypted before they leave this browser.

Browsers cannot truly prevent operating-system screenshots or screen recording. ANDO adds web-level deterrents and privacy shielding for testing, while native mobile/desktop apps can enforce stronger capture policies.

Security

End-to-end encryption

Message text is encrypted in the browser with a key derived from your private key and the recipient's public key. The backend stores only ciphertext, IV, sender, recipient, and expiry metadata.

Disappearing messages

Every message receives an expiry timestamp exactly 24 hours after creation. The server prunes expired messages on requests and a scheduled cleanup.

Capture blocking

Web apps can deter capture, hide content on blur, block print/context menus, and react to known keys. They cannot guarantee screenshot or screen-recording prevention across all devices.
